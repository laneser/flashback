# 11. 風險與技術債

## 11.1 技術風險矩陣

| ID | 風險 | 影響 | 機率 | 等級 | 緩解措施 |
|----|------|------|------|------|---------|
| R-1 | YAMNet 分類精度不足 | 無法區分飆車與正常車流 | 中 | 高 | AND 邏輯降低誤觸發；保留 fine-tuning 空間 |
| R-2 | 手機麥克風品質差異 | 不同手機偵測效果不一致 | 高 | 中 | 提供校準功能；使用相對音量而非絕對 dB |
| R-3 | Android 背景執行限制 | 系統殺死背景服務 | 中 | 高 | Foreground Service + WAKE_LOCK；電池優化白名單引導 |
| R-4 | 即時性要求難以達成 | 閃光延遲 > 100ms | 低 | 中 | 音訊管線優化；YAMNet 推論平行化 |
| R-5 | 長時間運作記憶體洩漏 | App crash | 中 | 中 | 定期 profiling；使用 LeakCanary |
| R-6 | WiFi 延遲影響多機同步 | 從機閃光延遲過大 | 中 | 低 | 接受 200~500ms 延遲；主機閃光不受影響 |
| R-7 | TarsosDSP 維護狀態 | 函式庫停止更新、API 不相容 | 低 | 低 | 必要時自行實作 FFT（算法成熟） |
| R-8 | iOS 雙平台維護成本 | 兩套程式碼需同步更新 | 中 | 中 | 核心邏輯相似；共通 WebSocket 協定確保互通性 |
| R-9 | Tart VM 僅支援 Apple Silicon | 非 Apple Silicon 開發者無法建置 iOS | 中 | 低 | iOS 開發為可選，Android 版本獨立運作 |

## 11.2 法律風險

| 風險 | 說明 | 緩解措施 |
|------|------|---------|
| 妨礙駕駛視線 | 直射閃光可能影響駕駛安全 | 安裝指南明確要求閃光朝向側牆反射，非直射駕駛眼睛 |
| 造成事故民事責任 | 若閃光導致交通事故 | 使用者自行承擔安裝責任；文件中加入免責聲明 |
| 隱私侵犯 | 自動拍照可能侵犯他人隱私 | 拍照功能為選配；照片僅供個人使用，不自動上傳 |
| 噪音法規誤解 | 使用者可能誤認系統具有法律舉證效力 | 明確標示系統定位為「嚇阻」而非「舉證蒐證」 |

## 11.3 技術債

| ID | 項目 | 說明 | 優先級 |
|----|------|------|--------|
| TD-1 | 無測試 | 目前無任何單元測試或 UI 測試 | 高 |
| TD-2 | 無 CI/CD | 缺少自動化建置與測試流程 | 中 |
| TD-3 | 硬編碼閾值 | 觸發條件閾值需移至 DataStore 設定 | 中 |
| TD-4 | 無 ProGuard/R8 規則 | Release 建置可能裁剪必要程式碼 | 低 |
| TD-5 | 無 crash 回報機制 | 正式環境 crash 無法追蹤 | 低 |
| TD-6 | 無國際化（i18n） | UI 文字僅支援繁體中文 | 低 |
| TD-7 | 無共用協定規格 | Android / iOS 間的 WebSocket JSON 協定尚未正式定義 | 中 |
