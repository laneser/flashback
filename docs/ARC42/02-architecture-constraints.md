# 2. 架構限制

## 2.1 技術限制

| 限制 | 說明 |
|------|------|
| Android 優先 | 目標平台為 Android（Min SDK 24 / Android 7.0）；iOS 版本計畫中（Swift + SwiftUI） |
| 離線 ML 推論 | 所有聲音分類必須在裝置端完成，不依賴雲端服務 |
| 即時音訊處理 | 音訊管線延遲需控制在 100ms 以內 |
| 手機硬體限制 | 閃光燈亮度、麥克風品質受限於手機硬體規格 |
| JDK 17 | 編譯目標為 Java 17 |
| Kotlin 2.0+ | 使用 Kotlin 2.0.21，搭配 K2 compiler |

## 2.2 組織限制

| 限制 | 說明 |
|------|------|
| 開源專案 | 程式碼公開，架構設計需便於社群理解與貢獻 |
| 單人起步 | 初期由個人開發，架構需簡潔可維護 |
| 低成本 | 硬體成本需壓低（利用舊手機），不使用付費 API |
| 繁體中文文件 | 文件以繁體中文撰寫，技術術語保留英文原文 |

## 2.3 法律限制

| 限制 | 說明 |
|------|------|
| 閃光安全 | 閃光不得直射駕駛視線，需朝向側牆反射 |
| 隱私保護 | 若拍照存檔，照片僅供個人使用，不公開上傳 |
| 定位為嚇阻 | 系統定位為「聲光嚇阻」而非「舉證蒐證」 |
| 安裝責任 | 安裝位置與角度由使用者自行負責 |

## 2.4 技術慣例

| 慣例 | 說明 |
|------|------|
| Gradle Version Catalog | 所有依賴版本集中於 `libs.versions.toml` |
| Kotlin DSL | 建置腳本使用 Kotlin DSL（`.gradle.kts`） |
| Jetpack Compose | UI 層使用 Compose，不使用傳統 XML layout |
| MVVM 架構 | 遵循 Android 官方推薦的架構模式 |
