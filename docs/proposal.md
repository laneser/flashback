# 🚨 飆車聲音偵測與閃光嚇阻系統
**專案提案 v1.0**  
日期：2026-02-22

---

## 一、專案背景與動機

台灣都市深夜飆車問題嚴重，尤其在隧道、高架、特定路段，改裝車輛炸街聲音超過 100dB，嚴重影響居民睡眠品質與公共安全。現有處理方式（檢舉、警察巡邏）效率低且缺乏即時性。

本專案目標是開發一套**低成本、自動化、民間可自行架設**的聲音偵測與閃光嚇阻裝置，利用飆車族對「被拍照」的心理恐懼達到嚇阻效果。

---

## 二、核心概念

麥克風持續監聽
    ↓
AI 分析聲音特徵（音量 + 頻譜 + 聲音分類）
    ↓
判斷為飆車聲
    ↓
立即觸發閃光燈（模擬測速照相）
    ↓
同步通知多支裝置聯動閃光

---

## 三、功能規格

### 3.1 聲音偵測
- 持續監聽環境音（低功耗模式）
- 即時計算音量（RMS dB）
- FFT 頻譜分析，擷取引擎聲特徵頻段（100Hz ~ 3kHz）
- 整合 **YAMNet**（Google TensorFlow Lite 模型）辨識聲音種類
  - 支援辨識：摩托車引擎、汽車引擎、輪胎聲、爆鳴聲（backfire）

### 3.2 觸發條件（AND 邏輯，降低誤觸發）
| 條件 | 參數（可調整） |
|------|--------------|
| 音量超過閾值 | 預設 85dB |
| YAMNet 信心度 | 引擎/摩托車類 > 70% |
| 持續時間 | 連續 0.3 秒以上 |
| 啟動時段 | 預設 22:00 ~ 05:00 |

### 3.3 閃光燈控制
- **主裝置**：Android 手機內建閃光燈，本機直接觸發（延遲 < 100ms）
- **從裝置**：透過 WebSocket 廣播，2~3 支手機聯動閃光（延遲 200~500ms）
- **外接閃光**（選配）：ESP32 或智慧插座控制攝影棚閃光燈，亮度更強

### 3.4 記錄與通知
- 觸發時自動拍照存檔（含時間戳記）
- 記錄觸發 log（時間、音量、頻譜截圖）
- 選配：推播通知到手機 / Telegram Bot 通報

## 五、技術 Stack

| 層次 | 技術選擇 | 說明 |
|------|---------|------|
| 語言 | Kotlin | 原生 Android，效能最佳 |
| 音訊輸入 | AudioRecord API | 低延遲 PCM 串流 |
| 頻譜分析 | TarsosDSP | 開源音訊分析函式庫 |
| 聲音分類 | YAMNet (TFLite) | Google 開源，521 種聲音，離線運作 |
| 相機/閃光 | CameraX | 現代 Android 相機 API |
| 網路同步 | Ktor WebSocket | 純 Kotlin，低延遲廣播 |
| 外接控制 | ESP32 + MicroPython | 選配，控制外部閃光設備 |
| UI | Jetpack Compose | 現代 Android UI |

## 六、硬體需求

### 最小配置（單機）
- Android 手機（Android 8.0+）× 1
- 固定支架（對準道路方向）
- 充電線（長時間運作需持續供電）

### 建議配置（多機聯動）
- 主手機 × 1（偵測 + 主閃光）
- 從手機 × 2~3（聯動閃光，製造多點效果）
- 同一 WiFi 網路或手機熱點

### 進階配置（最強嚇阻）
- 以上 + ESP32 開發板 × 1（NT$200）
- 攝影棚閃光燈或大功率 LED 頻閃燈 × 1~2
- 防水盒（戶外安裝用）

## 八、法律風險評估

| 風險 | 說明 | 降低方式 |
|------|------|---------|
| 妨礙駕駛視線 | 直射閃光可能影響駕駛 | 閃光朝向側牆反射，非直射駕駛眼睛 |
| 造成事故民事責任 | 若閃光導致事故 | 安裝位置避免正對車道，調整角度 |
| 隱私問題 | 若同時拍照存檔 | 照片僅供個人使用，不公開上傳 |

**建議：** 本系統定位為「聲光嚇阻」而非「舉證蒐證」，閃光以「讓飆車族以為被拍」為目的，安裝角度與亮度需謹慎規劃。
